Git常用命令

发表于 2016-10-13   |   分类于 tools   |  

Git是一款免费、开源的分布式版本控制系统，用于敏捷高效地处理任何或小或大的项目。
git作为常用版本管理工具，同svn一样也有GUI窗口操作，本文是对它几个常用的命令进行简单介绍，以下操作都是在安装好git的前提下，不涉及安装配置。主要介绍下面几个命令：

git clone

从远程服务器clone下一个版本库,我这儿远程服务器使用的是github,下载nodejs。
命令语法 git clone <版本库地址> [本地目录]

如下就开始下载

Cloning into ‘F:/repo/nodejs’…
remote: Counting objects: 252182, done.
remote: Compressing objects: 100% (36/36), done.
Receiving objects: 43% (110476/252182), 135.79 MiB | 329.00 KiB/s
在本地的目录 f 盘下repo/nodejs就可以看到已经下载好的代码。

git remote

用于管理远程仓库名称,类似别名，默认clone下来的时候代码仓库名称为origin，也可以通过git clone -o name url 来指定名称。

git fetch

从远程取回更新，如果不指定分支名，会默认取回所有的分支，可以通过branch命令查看分支，
取回后可以根据当前分支创建新的分支，也可以和本地分支进行合并

例子：

$ git branch -r
origin/HEAD -> origin/master
origin/archived-io.js-v0.10
origin/archived-io.js-v0.12
…

git push

本地做了更新，需要提交到远程服务器，可以在github创建一个仓库，然后做测试。

git push <远程代码库名称> <本地分支名>:<远程分支名>

切换到需要推送代码的目录，执行git push,如果省略分支名，则会对服务器进行对比，
如果存在就进行提交，不存在就新建分支。如果只有一个分支，远程代码库名称，本地分支都可以省略，
默认推送当前分支。如果不知道代码库名称和分支名都可通过命令查询。
注意： 如果指定了远程分支，没有指定本地分支，则推送一个空的分支进行替换远程分支，等同于删除分支

push 还有一些其他参数，本文只简单介绍常用的。

以上就是一个完整的提交流程，如果有冲突，需要先进行合并，在进行提交，也可以通过命令-force进行强制提交。

git pull

从某个分支进行更新，并且与本地进行合并
git pull [远程代码库名称] [远程分支名]:[本地分支名]
同push 一样，后面几个参数都可以省略，默认是当前分支

代码库名称，分支名可以省略是因为在clone的时候回自动建立追踪关系，也可以手动创建这种关系。

git 命令还有很多配置项，需要不断的学习。
