Weblogic

发表于 2016-10-13   |   分类于 java   |  

weblogic 作为企业常用的web服务器，提供了如分布式事务，jndi数据源，集群部署，性能监控等很实用的功能，之前项目使用weblogic双机部署，折腾了好久，这次把从安装到配置集群的步骤写下来，之前安装介质都是客户提供，机器是128G内存，硬盘1TB ,8核cpu 本文主要先介绍单机安装及一些配置。集群安装又不一样。

准备

一台基本环境安装好的linux虚拟机，weblogic安装介质。使用putty连接虚拟机。
weblogic 版本：Oracle WebLogic Server 11gR1 (10.3.6)
jdk版本：1.7

新的虚拟机没有安装Java，先去安装Java环境，Oracle官网找到下载地址。
wget http://download.oracle.com/otn-pub/java/jdk/7u79-b15/jdk-7u79-linux-x64.tar.gz

如果虚拟机不能联网，可以先下载好，通过putty 的pscp 命令或者其他方式将文件传到虚拟机上。
解压安装，配置好环境，我这儿配置的是全局环境，每个用户都可以使用。

在profile 文件末尾写入

export JAVA_HOME=/usr/local/jdk1.7.0_79

export PATH=$JAVA_HOME/bin:$PATH

export CLASSPATH=.:$JAVA_HOME/lib/dt.jar:$JAVA_HOME/lib/tools.jar

退出并且保存。
然后执行source /etc/profile 让环境变量立即生效。

创建文本weblogic用户组，等会就使用这个这个用户进行安装。

开始安装

安装方式有两种，1 通过界面安装 2 字符界面的静默安装 ，如果你的虚拟机没有安装界面程序，也要选择界面安装，可以通过xmnager软件的Xstart进行图形化安装，本文则主要介绍静默安装。
注意这个时候是weblogic用户登录。

weblogic控制台

已经配置完成，物理机访问 http://192.168.1.112:7001/console , 这个ip是虚拟机的ip，如果不能访问需要配置防火墙规则将7001端口开放出来。
输入刚刚配置好的用户名及密码就可以登录控制台进行管理weblogic
